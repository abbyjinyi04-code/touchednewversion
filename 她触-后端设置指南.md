# 她触 Touched · 后端设置指南（正式内测版 · 可支撑上百人）

这份指南帮妳把「她触」升级成**可以正式邀请内测、支撑上百人**的版本：

- ✅ **真实登录验证**：用 Firebase Authentication，密码由 Google 安全保管，我们的数据库里**不存任何密码**。
- ✅ **数据隐私**：私聊、深度链接、感受日记**只有当事人能读**，别人从数据库层面就读不到。
- ✅ **数据永久保存 + 跨设备**：同一手机号密码，手机电脑都能登，数据自动同步。
- ✅ **可扩展到上百人**：只按需读取「与妳相关」的数据，不会把所有人的数据全拉下来，省额度。
- ✅ **真实互动**：拉手、聊天、点赞评论、轻触、正式链接邀请，都是真人之间双向进行。

一次性设置约 **12 分钟**。免费额度（每天 5 万次读 / 2 万次写）对上百人内测足够。

> 没填配置时仍是「本地模式」正常运行，不会弄坏现有版本。

---

## 一、创建 Firebase 项目

1. 打开 https://console.firebase.google.com ，用 Google 账号登录。
2. **添加项目 / Add project** → 名字随便（如 `touched`）→ Google Analytics 可关掉 → 创建。

## 二、开启「登录方式」（重要！新步骤）

1. 左侧菜单 **构建 Build → Authentication（身份验证）** → 点 **「开始使用 / Get started」**。
2. 在 **Sign-in method（登录方式）** 里，找到 **「电子邮件/密码 Email/Password」** → 点开 → **启用第一项**（不用启用"邮件链接"）→ 保存。

> 我们用「手机号 + 密码」做界面，底层自动转成 `手机号@touched.app` 的邮箱登录——所以**不需要短信、不花钱**，姐妹们填手机号即可。

## 三、添加 Web 应用，拿配置

1. 项目主页点 **`</>`（Web）** 图标 → 取昵称 `touched-web` → **不要**勾 Hosting → 注册。
2. 复制它给的 `firebaseConfig`（apiKey、projectId 等几项）。

## 四、把配置粘进 index.html

1. 打开 `index.html`，搜索 `FIREBASE_CONFIG`。
2. 把每个 `PASTE_...` 换成 Firebase 给妳的真实值，保存。

## 五、创建数据库并设置「隐私规则」

1. 左侧 **构建 Build → Firestore Database → 创建数据库** → **测试模式开始** → 选近的地区（如 `asia-east1`）→ 创建。
2. 进入 **规则 / Rules** 标签，把内容**整段替换**成下面这套（这套规则保证：私聊/关系/日记只有当事人能读），然后 **发布 / Publish**：

```
rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {
    function signedIn() { return request.auth != null; }

    // 公开档案：登录后可见；只有本人能改
    match /users/{uid} {
      allow read: if signedIn();
      allow create, update: if signedIn() && request.auth.uid == uid;
      allow delete: if false;
    }
    // 私密数据（感受日记等）：只有本人能读写
    match /private/{uid} {
      allow read, write: if signedIn() && request.auth.uid == uid;
    }
    // 烦恼屋问题：登录可见；作者可建/删；登录者可更新（用于轻触）
    match /questions/{id} {
      allow read: if signedIn();
      allow create: if signedIn() && request.resource.data.authorId == request.auth.uid;
      allow update: if signedIn();
      allow delete: if signedIn() && resource.data.authorId == request.auth.uid;
    }
    // 派对屋帖子：登录可见；作者可建/删；登录者可更新（点赞/评论）
    match /posts/{id} {
      allow read: if signedIn();
      allow create: if signedIn() && request.resource.data.authorId == request.auth.uid;
      allow update: if signedIn();
      allow delete: if signedIn() && resource.data.authorId == request.auth.uid;
    }
    // 她意识故事：登录可见；作者可建/删；登录者可更新（看见🤍）
    match /stories/{id} {
      allow read: if signedIn();
      allow create: if signedIn() && request.resource.data.authorId == request.auth.uid;
      allow update: if signedIn();
      allow delete: if signedIn() && resource.data.authorId == request.auth.uid;
    }
    // 关系 + 私聊：只有两位当事人能读写，其她人读都读不到
    match /relationships/{id} {
      allow read, update, delete: if signedIn() && request.auth.uid in resource.data.members;
      allow create: if signedIn() && request.auth.uid in request.resource.data.members;
    }
    // 演示种子标记（不含任何敏感信息）
    match /meta/{doc} {
      allow read: if signedIn();
      allow write: if signedIn();
    }
  }
}
```

> 这套规则的关键：`relationships` 只有 `members`（两位当事人）能访问——**私聊内容在数据库层面就被锁住，别人技术上都拿不到**。日记放在 `private/{自己}`，同样只有本人能读。

## 六、重新部署到 GitHub Pages

1. 把改好的 `index.html` 重新上传 / 覆盖到妳的仓库（方法见上一条消息：`Add file ▸ Upload files` 拖进去，或点铅笔✏️直接改 `FIREBASE_CONFIG`）。
2. 等 1～2 分钟，打开网址，第一次加载会**自动写入演示数据**（示例姐妹 + 示例内容），让先进来的人不孤单。

- 电脑注册账号 → 发帖 / 发问。
- 手机打开同一网址 → **登录**（同一手机号密码）→ 应能看到内容、资料同步。✅
- 两个账号互相「拉手」→ 双方在「触碰」里都能看到这段对话，能真实聊天。✅

## 八、能撑多少人？

- 免费额度：**每天 5 万次读、2 万次写、1 GB 存储**。
- 这一版只按需读取「与妳相关」的数据 + 公开内容限量加载，所以**几十到上百人内测，免费额度通常够用**。
- 如果以后人更多 / 很活跃，担心触顶，可在 Firebase 开 **Blaze 按量付费**并设预算上限——这种规模一般每月也就几毛到几块钱；开了之后没有每日硬上限。

## 九、隐私与安全说明

- **密码**：由 Firebase Authentication 安全管理，数据库里不存密码。✅
- **私聊 / 关系 / 日记**：规则限制只有当事人能读，其她人读不到。✅
- **公开内容**（提问、帖子、故事）：本来就是给大家看的，所有登录用户可见——这是产品设计，不是漏洞。
- **管理员口令**：存在本机、不上云（避免任何人读到）。默认 `touched-admin`，进控制台后可改；想让信任的姐妹也能用，把口令告诉她即可，或在控制台加额外口令（按设备生效）。
- 邀请制内测建议：在邀请群里说明这是内测产品，请勿在平台上填写高度敏感的个人隐私。

## 十、管理员（开发者控制台）

- 入口：`网址/#dev`，或登录后在「我的」页底部「🛠️ 开发者入口」。口令 `touched-admin`。
- 功能：概览（数据/健康/自检）、运行日志（自动抓报错）、内容审核（隐藏对全员即时生效；删除仅能删自己创建的，删别人请到 Firebase 控制台）、数据导出备份。
- 注意：云端模式下，管理员也需要**先登录一个账号**才能读取云端数据做审核。

## 十一、常见问题

**填了配置还是要重新注册？** GitHub 传的多半还是旧文件，或浏览器缓存——确认上传的是改过配置的版本，并强制刷新 / 用无痕窗口。

**登录报错 `auth/operation-not-allowed`？** 第二步「Email/Password 登录方式」没启用，回去开启。

**数据不同步 / 报错？** `网址/#dev` →「运行日志」看红色错误，常见是配置贴错或规则没发布。把报错发我即可。
